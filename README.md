## 📚 Теоретическая часть: Что мы будем изучать?

### Что такое Wireshark и зачем он нужен?

**Wireshark** — это программа-анализатор сетевого трафика (сниффер), которая позволяет просматривать все данные, передаваемые по сети в реальном времени. Представь, что ты можешь заглянуть в каждый конверт (пакет), который отправляется по сети, и прочитать его содержимое.

Программа работает, переводя сетевую карту компьютера в **неразборчивый режим (promiscuous mode)** — это специальный режим, при котором сетевая карта принимает ВСЕ пакеты, проходящие через сеть, а не только те, что адресованы ей.

**Основные возможности Wireshark:**
- **Захват live-трафика** в реальном времени
- **Глубокая инспекция пакетов** — программа "знает" структуру сотен сетевых протоколов
- **Фильтрация** — возможность отсеивать ненужные пакеты и оставлять только интересующие
- **Реконструкция сессий** — можно восстановить весь разговор между клиентом и сервером
- **Экспертная система** — Wireshark сам подсвечивает потенциальные проблемы

### Почему мы используем HTTP, а не HTTPS?

**HTTP (HyperText Transfer Protocol)** передает данные в открытом, незашифрованном виде. Это значит, что любой, кто имеет доступ к сети, может прочитать:
- Логины и пароли
- Содержимое страниц
- Cookies и сессионные данные

**HTTPS (HTTP Secure)** добавляет шифрование (SSL/TLS), и данные становятся нечитаемыми для посторонних.

В нашем учебном проекте мы намеренно используем HTTP, чтобы наглядно увидеть, как происходит перехват.

**Как это работает:**
1. Жертва (Windows) заходит на сайт, размещенный на Ubuntu Server
2. Вводит логин и пароль в форму на сайте
3. Данные отправляются методом POST (в открытом виде)
4. Kali Linux, находящийся в той же сети, перехватывает эти пакеты
5. Wireshark на Kali показывает содержимое пакетов, включая пароль

## 🔧 Пошаговая инструкция по развертыванию

### Шаг 0: Подготовка VirtualBox и Host-Only сети

**Что такое Host-Only сеть?** 
Это виртуальная сеть, которая соединяет только твои виртуальные машины между собой и с хост-компьютером. Выхода в интернет через эту сеть нет (если не настроить дополнительно), что делает её идеальной для безопасных экспериментов.

**Создание Host-Only адаптера:**

1. Открой **Oracle VM VirtualBox**
2. В главном меню выбери: **Файл** → **Инструменты** → **Network Manager** (или **Управление** → **Настройки** → **Сеть** в старых версиях)
3. Перейди на вкладку **Host-only Networks**
4. Нажми кнопку **Create** (зеленый плюсик)
5. Должен появиться адаптер с именем **vboxnet0**
6. Выдели его и нажми кнопку **Properties** (шестеренка)
7. Убедись, что параметры такие:
   - **IPv4 Address:** `192.168.56.1`
   - **IPv4 Network Mask:** `255.255.255.0`

### Шаг 1: Настройка виртуальных машин

#### Для Ubuntu Server (веб-сервер):

1. Выбери виртуальную машину Ubuntu Server
2. Нажми **Настроить** (или правой кнопкой → **Настроить**)
3. Перейди в раздел **Сеть**
4. Настрой два адаптера:
   
   **Адаптер 1:**
   - Включить сетевой адаптер: ☑️
   - Тип подключения: **NAT**
   - *Причина: Этот адаптер нужен для доступа в интернет (чтобы скачивать пакеты, Docker и т.д.)*
   
   **Адаптер 2:**
   - Включить сетевой адаптер: ☑️
   - Тип подключения: **Виртуальный адаптер хоста (Host-only)**
   - Имя: **vboxnet0**

#### Для Kali Linux (атакующий):

Абсолютно такая же конфигурация сети:
- **Адаптер 1:** NAT (для доступа к интернету и обновлений)
- **Адаптер 2:** Host-Only (vboxnet0) для полигона

#### Для Windows (жертва):

- **Адаптер 1:** Host-Only (vboxnet0) — ей не нужен интернет, только доступ к нашему сайту
- NAT можно отключить или оставить, но для чистоты эксперимента оставь только Host-Only

Шаг 5: Установка и настройка Wireshark на Kali Linux
Теория: что мы сейчас установим?
Wireshark состоит из двух частей:

Захватывающий модуль (основан на libpcap) — работает с правами root, потому что доступ к сетевым интерфейсам — привилегированная операция

Графический интерфейс — может работать от обычного пользователя

Установка:
bash
# Обновляем список пакетов
sudo apt update

# Устанавливаем Wireshark
sudo apt install wireshark -y
Важный момент! Во время установки появится окно с вопросом:

text

Выбери <Yes> — это позволит обычным пользователям (не root) захватывать трафик.

Добавление пользователя в группу wireshark:

bash
sudo usermod -aG wireshark $USER
Применяем изменения:

bash
# Либо выйди из системы и зайди снова
# Либо выполни:
newgrp wireshark
Запуск Wireshark:

bash
# Можно через меню (Интернет -> Wireshark)

wireshark &

Часть 3: Поиск и анализ перехваченных данных в Wireshark
Полезные фильтры Wireshark:

Фильтр	Что показывает
http	Весь HTTP трафик
http.request.method == "POST"	Только POST запросы
ip.src == 192.168.56.30	Все пакеты с IP жертвы
Как найти пароль:

Введи фильтр: http.request.method == "POST"

Найди пакет с методом POST в списке

Выдели этот пакет

В средней панели раскрой раздел Hypertext Transfer Protocol

Найди HTML Form URL Encoded

Раскрой — увидишь поля username и password с их значениями!

Пример того, что ты увидишь:

text
Hypertext Transfer Protocol
    POST /login.php HTTP/1.1\r\n
    HTML Form URL Encoded: application/x-www-form-urlencoded
        Form item: "username" = "admin"
        Form item: "password" = "superSecretPassword123"
